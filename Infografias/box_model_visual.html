Box model

Las capas que forman cada caja en CSS y cómo afectan al tamaño, espacios y desbordes: Tamaño (width/height, min/max-*), Espacios (margin, padding), Borde (border, border-radius), Caja (box-sizing) y Desbordes (overflow, outline).

1) Esquema del Box Model

Una caja está compuesta por content → padding → border → margin. El outline es un contorno externo para accesibilidad que no ocupa espacio.

margin
border
padding
Content

2) Tamaño: width, height, min/max-*

width/height definen el tamaño preferido. Con min-* y max-* marcas límites flexible/fluido.

width: clamp(240px, 60%, 780px)
width: 420px
.box { width: 420px; height: 160px; }
.box-fluid { width: clamp(240px, 60%, 780px); min-height: 120px; max-height: 70vh; }

3) Espacios: margin vs padding

margin es espacio externo (separa de otras cajas). padding es espacio interno (separa el borde del contenido).

Padding (espacio dentro)

Contenido con padding: el fondo crece hacia dentro.

Margin (espacio fuera)

Caja con margin: separa esta caja de su contenedor/hermanos.

Colapso de márgenes verticales: los márgenes adyacentes de elementos de bloque se “fusionan” (toma el mayor). Para evitarlo, añade padding, border o crea un nuevo contexto (p. ej., overflow: auto o display: flow-root).

4) Borde: border y border-radius

El borde rodea el padding y el contenido. border-radius puede ser circular (50%) o elíptico (usando / para radios separados en X/Y).

solid
dashed
double
.pill { border: 2px solid #49b9ff; border-radius: 999px; }
.badge { border: 2px dashed #49b9ff; border-radius: 12px; }

5) Caja: box-sizing: border-box;

Con content-box (por defecto), el ancho declarado no incluye padding ni border. Con border-box, el ancho anunciado sí los incluye (más predecible para layout).

content-box (default)

width:260 + padding(20*2) + border(6*2) ➜ caja total mayor

El total crece al sumar padding y border.

border-box (recomendado)

width:260 incluye padding y border ➜ caja total estable

Más fácil de razonar. Suele ponerse global: *{ box-sizing:border-box }.

* { box-sizing: border-box; } /* recomendado */
.card { width: 260px; padding: 20px; border: 6px solid #49b9ff; }

6) Desbordes: overflow y outline

overflow decide qué pasa si el contenido sobrepasa la caja. outline dibuja un contorno por fuera (no ocupa espacio), útil para focus y accesibilidad.

visible → El contenido puede sobresalir… ➜ ➜ ➜ ➜ ➜
hidden → Se recorta el exceso sin barras.
scroll → Muestra barras siempre.
auto → Barras solo si hacen falta.
clip → Recorte más estricto que hidden.
Enlace accesible (outline) 
.card:focus { outline: 3px solid white; outline-offset: 4px; } /* no altera layout */
.scroll-x { overflow-x:auto; overflow-y:hidden; }

7) Snippets útiles

/* Tamaño fijo vs fluido */
.box { width: 420px; height: 160px; }
.box-fluid { width: clamp(240px, 60%, 780px); min-height: 120px; max-height: 70vh; }

/* Espacios */
.pad { padding: 1rem; }
.marg { margin: 1rem 0; }

/* Borde y radios */
.badge { border: 2px dashed #49b9ff; border-radius: 12px; }
.pill  { border-radius: 999px; }

/* Caja global */
* { box-sizing: border-box; }

/* Desbordes y contorno */
.clip-1l { white-space: nowrap; overflow: hidden; text-overflow: ellipsis; }
.btn:focus { outline: 2px solid #fff; outline-offset: 2px; }

Recuerda: margin separa por fuera, padding separa por dentro; el borde va entre ambos; y con border-box la vida es más sencilla al dimensionar.
